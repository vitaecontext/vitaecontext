# X profile and post recipes

## Display-name recipe

Use the display name to improve immediate clarity when it helps the account's real positioning.

Default pattern:

`Name | role, niche, or outcome`

Rules:

- keep the actual person or brand name recognizable
- add a role or niche only when it improves clarity rather than clutter
- avoid stuffing multiple disconnected keywords into the display name

## Bio recipe

Treat the official 160-character bio limit as a compact landing page constraint.

Default structure:

`Who I help + How I help + Proof + CTA`

Writing rules:

- place the most important niche terms early
- keep it natural rather than keyword-stacked
- use real proof only: role, domain, shipped work, results, or recognized expertise
- use the CTA only when there is a real next step such as a link, resource, or pinned asset

## Pinned-post recipe

Use a pinned post only when there is a strong evergreen asset.

Good pinned-post candidates:

- a flagship thread
- a core case study
- a useful free resource
- a portfolio or repo explainer

Default pinned-post structure:

1. what the asset is
2. why it matters
3. what the reader will get
4. clear next step or link

If no strong evergreen post exists yet, leave the slot unpinned or rotate only after stronger material exists.

## Single-post recipe

Default structure:

1. strongest hook or topic first
2. one concrete insight, observation, or result
3. optional proof or example
4. optional open-ended question

Writing rules:

- prioritize clarity over cleverness
- avoid hollow engagement bait
- if an external link matters, make sure the post still delivers native value
- keep formatting mobile-readable with short paragraphs and line breaks

## Thread recipe

Use a thread when the topic genuinely needs depth, not as a default ranking tactic.

Default structure:

1. opening hook
2. problem or context
3. 3-6 supporting posts with concrete detail
4. outcome, takeaway, or synthesis
5. optional question to invite discussion

Thread rules:

- keep each post individually readable
- maintain one topic through the thread
- do not stretch a short idea into a thread just to look substantial

## Alt Text recipe

Alt Text is for accessibility first and AI interpretation second.

Default structure:

1. what the image shows
2. relevant context
3. important text, chart meaning, or data point if visible

Rules:

- describe the actual visual accurately
- include keywords only when they are genuinely part of the image context
- avoid filenames, placeholders, or vague labels

## Rewrite posture

When rewriting an X profile or post:

- preserve verified positioning and proof
- remove hype, vagueness, and keyword stuffing
- improve scannability and conversion intent
- keep the content aligned with the user's actual niche and posting capacity
