# Why and when to use a personal career context file

## Core purpose

A personal career context file is the factual source of truth for career-oriented agent work. It prevents the agent from starting from zero on every session and keeps outputs grounded in one maintained record.

## When it is most valuable

- the user wants repeated CV, LinkedIn, portfolio, or cover-letter work
- the user has many projects, roles, courses, or certifications
- different public surfaces are drifting out of sync
- the user wants agent outputs to stay consistent over time

## What it replaces poorly

- repeated long prompts from scratch
- copy-pasted raw exports with weak structure
- opaque memory features used as the main source of truth

## Decision rule

Start with context optimization first when:

- facts are messy
- dates and titles conflict across sources
- the task spans multiple public surfaces

If the user only needs a quick single-surface edit and facts are already clean, full context-file work can be optional.
