# CV section recipes

## Core section order

Default order:

1. contact information at the top of the document body
2. professional summary
3. experience
4. skills
5. education

Add other sections only when the user's profile genuinely requires them, such as projects, certifications, publications, or awards.

## Contact block recipe

- first line: full name only
- second line: location, email, phone if used, and explicit profile URLs
- keep all contact data in the document body, not the header or footer
- write URLs directly instead of hiding them behind anchor text

## Professional summary recipe

Use 2 to 4 lines.

Recommended flow:

1. target role or current role focus
2. strongest domain or stack fit
3. one or two evidence-backed strengths
4. optional specialization aligned to the job description

Rules:

- use plain language, not branding fluff
- mirror important hard-skill phrasing from the job description when true
- do not dump a list of disconnected keywords

## Experience recipe

For each entry:

- role title
- employer
- location if useful
- unambiguous date range
- 2 to 5 bullets focused on relevance and impact

Bullet pattern:

```text
Action + task or scope + tool or method + measurable result
```

Rules:

- start with an action verb
- use active voice
- keep each bullet grounded in real work
- prefer fewer strong bullets to many weak ones
- translate duties into achievements

## Skills recipe

- use a compact list of standard skill names
- prioritize hard skills, tools, platforms, and domain terms
- group loosely only if the grouping stays easy to parse
- keep the skills list aligned with the evidence in Experience and Education
- drop weak or unsupported skill claims

## Education recipe

- degree name
- institution
- location if useful
- date range or graduation date
- optional grade or honors only when relevant and accurate

For early-career profiles, education can carry more detail. For experienced profiles, compress it unless the degree is directly relevant to the target role.

## Section rewrite posture

- preserve facts and strengthen evidence density
- when exact numbers are unknown, use careful wording instead of fake precision
- keep section headers standard and easy for parsers to classify
