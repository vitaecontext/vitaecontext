# X Premium and confidence rules

## Premium rules

- Verify current official X documentation or visible account settings before giving paid-tier or capability-specific advice.
- Verify whether the user actually has Premium before suggesting Premium-only tactics.
- Premium can change product capabilities such as longer posts, longer video, reply prioritization, Boost access, and formatting.
- Premium features can vary by tier, platform, geography, account state, and current rollout.
- Treat Premium as additional leverage, not as a viral-growth guarantee.
- Treat paid Boost tools as separate from organic ranking behavior.
- If official verification is unavailable, label paid-tier guidance as historical or inferred.

## Long-form guidance

- Verify the current long-post limit before drafting beyond 280 characters.
- Treat long posts like mini-articles.
- Use strong opening lines and clear sectioning.
- Keep readability high; long-form does not excuse rambling.

## Reply-prioritization guidance

- Use any reply visibility advantage for high-value commentary.
- Do not farm low-effort replies for exposure.
- Apply reply-priority advantages to niche-relevant discussions, not random high-volume threads.

## Confidence labeling

Split advice into:

1. officially documented product capabilities
2. current recommender-system documentation
3. historical or open-source architectural inference
4. empirical tactics that should be tested

Do not blur those categories together.
