# Certification node workflow

Load this workflow for professional certifications, certificates, bootcamps, independent courses, workshops, or non-degree training.

Store records at `certifications/<credential-slug>.md`. Use `type: certification` and `id: certification:<stable-slug>`.

Capture issuer, official credential name, issue and expiration dates, credential ID, public URL, assessed scope, delivery format, practical requirements, demonstrated knowledge, and related applied work. Do not expose private credential identifiers unless the user wants them stored.

Keep university coursework inside the relevant degree subtree. Do not move it into certifications merely because the source calls it a course.

Before finishing, distinguish attendance certificates from assessed credentials and avoid treating either as proof of professional experience.
